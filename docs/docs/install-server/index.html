服务端部署

创建 docker-compose.yml 文件​

version: '3.9'

services:

  anynat:
    image: rockey1997/anynat:1.0.1 #镜像版本
    container_name: anynat_1 #容器名称
    ports:
      - "2333:2333" #服务端与客户端通信的端口  如果没有域名的情况下 可以使用 公网IP:2333进行连接
      - "443:443" #使用该端口需要绑定域名  例如a.com对应客户端80服务端口  b.com对应客户端服务端口81 如果使用IP+端口的访问方式 需要在此暴露对应的端口  例如bindPort是11300   就要添加11300:11300
      - "10000:22" #容器ssh端口
    restart: always
    volumes:
      - "./container/anynat/logs:/root/anynat/logs/pm2/logs" #日志存放路径
      - "./container/anynat/userConfig:/root/anynat/userConfig" #配置文件存放路径
    environment:
      VITE_RUNTIME_TYPE: server #运行方式 server:服务端 client:客户端

创建Anynat服务端容器​

docker compose up -d

进入配置文件存放文件夹​

cd ./container/anynat/userConfig

创建服务端连接配置文件: config.customize.ts ​

所有可配置的参数说明:https://github.com/rockey2020/anynat/blob/main/config.base.ts 

const customizeConfig = {
    server: {
        mode: "wss",
        bindDomains: ["example-anynat.next-app.cc"],//您的服务端和客户端的通信域名
    },
    client: {
        //需要穿透的内网服务列表(该项配置需要完全和客户端的配置一致)
        connections: [
            {
                type: "tcp",//内网服务协议类型 http https websocket ssh等 则填写tcp 其他根据实际情况填写udp 
                serverUrl: "wss://example-anynat.next-app.cc",//您的服务端和客户端的通信域名
                //uuid不能有重复 每一个服务对应独立的uuid
                uuid: "3ab16a66-1736-4981-bd6a-a0d4a7a2ee07",
                secretKey: "Qdc5q7fGr*YJ82TvwsrM",
                localPort: 11300,//内网本地服务的端口
                localHost: "192.168.124.114",//内网本地服务的IP
                bindPort: 11300,//该内网服务绑定的对应服务端端口 如果没有域名的话 也可以通过 服务端公网IP:11300 访问到您要穿透的内网服务  使用IP+端口访问内网服务的时候  记得在服务端docker-compose.yml文件暴露对应的端口
                bindDomains: ["example-a1.next-app.cc"],//配置好后 访问example-a1.next-app.cc 就能访问到你的内网服务
                //请到首页生成新的AES加密信息,并且分别覆盖 aesKey iv 对应的值 切莫使用本文档中用作演示的aes加密信息
                encryption: {
                    aesKey: {
                        "alg": "A256GCM",
                        "ext": true,
                        "k": "RGKoHaLq86ghkmBtTwiDeOnx0gQLB_vhS8eiF39xYUo",
                        "key_ops": ["encrypt", "decrypt"],
                        "kty": "oct"
                    },
                    iv: new Uint8Array([178, 212, 14, 44, 198, 193, 74, 134, 200, 107, 69, 144, 133, 24, 198, 254, 150, 118, 224, 96, 57, 135, 131, 113, 164, 34, 55, 19, 79, 246, 11, 185]),
                },
                hasSSL: false,//如果该内网服务启用了ssl功能,则该选项需要配置为true,否则无法使用内网穿透功能
                rejectUnauthorized: false,//配置false 忽略掉证书相关错误(服务端和客户端之间的通信)
            },
        ],
    },
};

export {customizeConfig};

重启容器,每次修改配置都需要重启容器/重启系统​

docker restart anynat_1

至此,恭喜你完成服务端部署安装教程​
